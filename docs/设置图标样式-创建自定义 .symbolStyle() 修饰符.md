# 设置图标样式-创建自定义 `.symbolStyle()` 修饰符

下面是SwiftUI中系统图标的常规用法：

```swift
import SwiftUI

struct SymbolRenderingModeView: View {
    
    var body: some View {
        VStack {
            // .monochrome 单色
            Image(systemName: "sun.rain.fill")
                .font(.system(size: 100))
                .symbolRenderingMode(.monochrome)
                .foregroundStyle(Color.cyan)
            
            // .hierarchical 分层（同一色系的分层渲染，自动衍生主次色）
            Image(systemName: "sun.rain.fill")
                .font(.system(size: 100))
                .symbolRenderingMode(.hierarchical)
                .foregroundStyle(Color.cyan)
            
            // .palette 调色板（每层一个指定颜色）
            Image(systemName: "sun.rain.fill")
                .font(.system(size: 100))
                .symbolRenderingMode(.palette)
                .foregroundStyle(Color.green, Color.blue)
            
            // .multicolor 多色（使用符号自带的内建颜色）
            Image(systemName: "sun.rain.fill")
                .font(.system(size: 100))
                .symbolRenderingMode(.multicolor)
        }
    }
}

#Preview {
    SymbolRenderingModeView()
        .preferredColorScheme(.dark)
}
```

要实现类似 `.symbolStyle()` 的自定义修饰符，我们可以创建一个 View 的扩展，封装 SwiftUI 的符号渲染逻辑。以下是完整的实现方案：

## 1. 定义符号样式枚举和修饰符

首先，创建一个枚举来表示不同的符号样式：

```swift
enum SymbolStyle {
    case monochrome(color: Color)
    case hierarchical(color: Color)
    case palette([Color])
    case multicolor
}
```

## 2. 创建 View 扩展实现修饰符

```swift
extension View {
    func symbolStyle(_ style: SymbolStyle) -> some View {
        Group {
            switch style {
            case .monochrome(let color):
                self
                    .symbolRenderingMode(.monochrome)
                    .foregroundStyle(color)
                
            case .hierarchical(let color):
                self
                    .symbolRenderingMode(.hierarchical)
                    .foregroundStyle(color)
                
            case .palette(let colors):
                self
                    .symbolRenderingMode(.palette)
                    .foregroundStyle(colors.first ?? .primary, colors.dropFirst().first ?? .primary)
                
            case .multicolor:
                self
                    .symbolRenderingMode(.multicolor)
            }
        }
    }
}
```

## 3. 使用示例

现在你可以像这样使用自定义修饰符：

```swift
struct ContentView: View {
    var body: some View {
        VStack(spacing: 30) {
            Image(systemName: "sun.rain.fill")
                .font(.system(size: 100))
                .symbolStyle(.palette([.green, .blue]))
            
            Image(systemName: "sun.rain.fill")
                .font(.system(size: 100))
                .symbolStyle(.monochrome(color: .cyan))
            
            Image(systemName: "sun.rain.fill")
                .font(.system(size: 100))
                .symbolStyle(.hierarchical(color: .cyan))
            
            Image(systemName: "sun.rain.fill")
                .font(.system(size: 100))
                .symbolStyle(.multicolor)
        }
    }
}
```

## 4. 完整实现（带默认值）

如果你想要更简洁的 API，可以添加一些便捷初始化方法：

```swift
extension SymbolStyle {
    static func palette(_ colors: Color...) -> SymbolStyle {
        .palette(colors)
    }
    
    static func monochrome(_ color: Color = .primary) -> SymbolStyle {
        .monochrome(color: color)
    }
    
    static func hierarchical(_ color: Color = .primary) -> SymbolStyle {
        .hierarchical(color: color)
    }
}
```

这样使用起来更加简洁：

```swift
Image(systemName: "sun.rain.fill")
    .font(.system(size: 100))
    .symbolStyle(.palette(.green, .blue))
```

## 5. 注意事项

1. 对于调色板模式(`.palette`)，我们只取了前两个颜色，因为大多数 SF Symbols 最多支持两层颜色
2. 如果传入的颜色不足，会使用 `.primary` 作为默认值
3. 多色模式不需要任何颜色参数
4. 这个修饰符可以应用于任何 View，但主要设计用于 `Image(systemName:)`

这个自定义修饰符封装了 SwiftUI 的符号渲染逻辑，提供了更简洁、类型安全的 API，同时保持了原始功能的灵活性。
